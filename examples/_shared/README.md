# `examples/_shared/`（示例库共享资产）

本目录用于存放 `examples/**` 多个示例之间可复用的**共享资产**与**小型固定测试数据**，避免复制粘贴导致漂移。

## 允许放什么

- 小体积、可公开的静态文件（例如：`*.txt`、`*.md`、`*.json`、`*.csv`、小图片）。
- 示例需要的“确定性输入数据”（用于离线回归/可复现演示）。
- 可复用的片段（例如：通用规则样例、模板片段），但应保持通用，不写死业务名词。

## 不允许放什么

- 任何密钥、token、个人数据、或需要脱敏的真实数据集。
- 大文件（避免膨胀仓库体积）。如确需大文件，应改为生成脚本（离线可生成）或用 stub。
- 只对单个示例有意义的产物（这类应留在各自示例目录的 `_artifacts/` 或等价目录中，并在 `.gitignore` 排除）。

## 约定（建议）

- 按用途分子目录，例如：
  - `datasets/`：示例输入数据（csv/json）
  - `fixtures/`：固定测试夹具
  - `images/`：图片资源（尽量小）
- 文件名表达语义，避免 `data1.csv` 这类含糊命名。
- 如果某个示例引用了共享资产，应在该示例的 `README.md` 中注明引用路径与用途。
